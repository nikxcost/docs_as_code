# Руководство по началу работы с документацией

Это руководство поможет вам настроить рабочее окружение для работы с документацией компании.

## Что вам понадобится

- Компьютер с Windows или macOS
- Аккаунт GitHub (попросите добавить вас в collaborators репозитория)
- Personal Access Token для GitHub (инструкция ниже)

## Шаг 1: Создание Personal Access Token

GitHub требует токен вместо пароля для синхронизации. Создайте его один раз:

1. Откройте [github.com/settings/tokens](https://github.com/settings/tokens)
2. Нажмите **Generate new token (classic)**
3. Введите название (например, "Obsidian")
4. Выберите срок действия (рекомендуется "No expiration")
5. Поставьте галочку на **repo** (полный доступ к репозиториям)
6. Нажмите **Generate token**
7. **Скопируйте токен и сохраните его** — он покажется только один раз!

## Шаг 2: Скачивание репозитория

1. Откройте репозиторий: [github.com/nikxcost/docs_as_code](https://github.com/nikxcost/docs_as_code)
2. Нажмите зелёную кнопку **Code**
3. Выберите **Download ZIP**
4. Распакуйте архив в удобную папку (например, `Documents/docs_as_code`)

## Шаг 3: Установка Obsidian

1. Скачайте Obsidian: [obsidian.md](https://obsidian.md/)
2. Установите приложение
3. При первом запуске выберите **Open folder as vault**
4. Выберите папку `docs` внутри скачанного репозитория (например, `Documents/docs_as_code/docs`)
5. На вопрос "Trust author and enable plugins?" нажмите **Trust author and enable plugins**

!!! success "Готово!"
    Все плагины и настройки загрузятся автоматически из репозитория.

## Шаг 4: Настройка синхронизации

При первой попытке синхронизации Obsidian запросит авторизацию:

1. Нажмите `Ctrl+P` (или `Cmd+P` на Mac)
2. Введите "git" и выберите **"Git: Commit all changes"** (или "Git: Push")
3. При запросе авторизации введите:
   - Username: ваш логин GitHub
   - Password: **ваш Personal Access Token** (не пароль!)

!!! note "Автосинхронизация"
    После авторизации ваши изменения будут автоматически синхронизироваться каждые 5 минут. Изменения коллег также загружаются автоматически.

## Шаг 5: Начало работы

### Структура документации

```
products/
└── E001 Название продукта/          # Продукт
    ├── index.md                      # Описание продукта
    └── FB001 Функциональный блок/    # Функциональный блок
        ├── index.md                  # Описание блока
        └── F001 Функция/             # Функция
            ├── index.md              # Описание функции
            ├── requirements.md       # Требования
            └── process.md            # Процесс
```

### Создание новых элементов

Используйте команды QuickAdd для создания структуры документации:

1. Нажмите `Ctrl+P` (или `Cmd+P` на Mac)
2. Введите "QuickAdd" и выберите нужную команду:

| Команда | Что создаёт |
|---------|-------------|
| **QuickAdd: Создать Продукт (Exxx)** | Новый продукт с полной структурой (блок + функция) |
| **QuickAdd: Создать Функц. блок (FBxxx)** | Новый функциональный блок в выбранном продукте |
| **QuickAdd: Создать Функцию (Fxxx)** | Новую функцию в выбранном блоке |

!!! tip "Формат ввода"
    При создании вводите номер и название через пробел, например: `001 CRM Оператора`

### Редактирование документов

1. Откройте нужный файл в дереве слева
2. Редактируйте текст как обычно
3. Нажмите `Ctrl+S` для сохранения
4. Изменения автоматически синхронизируются через 5 минут

### Добавление картинок

1. Скопируйте изображение в буфер обмена (или сделайте скриншот)
2. В Obsidian нажмите `Ctrl+V`
3. Картинка автоматически сохранится в папку `assets/images/`

### Добавление тегов к документам

Теги помогают группировать связанные документы и улучшают навигацию.

Добавьте теги в начало любого markdown-файла:

```yaml
---
tags:
  - CRM
  - требования
  - draft
---

# Заголовок страницы
```

**Рекомендуемые теги:**

| Тег | Когда использовать |
|-----|-------------------|
| `draft` | Документ в работе, не финальная версия |
| `ready` | Документ готов и актуален |
| `deprecated` | Устаревший документ |
| `требования` | Страница с требованиями |
| `процесс` | Описание бизнес-процесса |

Все теги собраны на странице [Теги](tags.md).

!!! tip "Теги в Obsidian"
    В Obsidian теги также работают — можно искать по тегам через поиск.

## Возможности сайта документации

На сайте документации доступны следующие возможности:

### Информация о документе

Внизу каждой страницы отображается:

- **Дата последнего обновления** — видно насколько актуален документ (например, "2 часа назад")
- **Дата создания** — когда документ был впервые добавлен
- **Авторы** — кто работал над документом (на основе истории изменений)

### Навигация

- **Мгновенные переходы** — страницы загружаются без перезагрузки
- **Оглавление справа** — следит за вашим скроллом
- **Кнопка "Наверх"** — быстрый возврат к началу страницы

### Поиск

- Нажмите `/` или кликните на поиск
- Поддерживает русский язык
- Можно поделиться ссылкой на результаты поиска

### Копирование кода

В блоках кода появляется кнопка копирования — один клик и код в буфере обмена.

## Частые вопросы

### Как посмотреть актуальную документацию?

Откройте в браузере: [https://nikxcost.github.io/docs_as_code/](https://nikxcost.github.io/docs_as_code/)

### Как получить последние изменения коллег?

Изменения загружаются автоматически. Для ручной загрузки:
`Ctrl+P` → "Git: Pull"

### Как немедленно отправить изменения?

`Ctrl+P` → "Git: Commit-and-sync"

### Что делать при конфликте?

Если вы и коллега редактировали один файл одновременно:

1. Появится уведомление о конфликте
2. Откройте файл — конфликтные места будут отмечены
3. Выберите правильную версию текста
4. Сохраните файл

При сложных конфликтах обратитесь к тимлиду.

### Почему я не вижу некоторые папки?

Служебные папки (scripts, templates, .github) скрыты от аналитиков, чтобы не отвлекать от работы с документацией. Это нормально.

### Команды QuickAdd не работают

1. Убедитесь, что плагин QuickAdd установлен и включён
2. Перезапустите Obsidian
3. Если не помогло — обратитесь к тимлиду
