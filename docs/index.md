# Product Documentation

Добро пожаловать в документацию продуктов компании.

## Структура документации

Документация организована по следующей иерархии:

```
Product → Capability → Feature → Requirements
```

- **Product** — продукт компании (например, Product A)
- **Capability** — функциональный блок продукта (например, Payments)
- **Feature** — конкретная функция (например, Refund)
- **Requirements** — детальные требования к функции

## Быстрый старт

### Для Business Analyst

1. Откройте этот репозиторий в Obsidian как Vault
2. Используйте шаблоны из папки `templates/` для создания новых документов
3. Вставляйте скриншоты через Ctrl+V — они автоматически сохранятся в `assets/images/`

### Просмотр документации

Для локального просмотра:

```bash
mkdocs serve
```

Откройте в браузере: [http://localhost:8000](http://localhost:8000)
